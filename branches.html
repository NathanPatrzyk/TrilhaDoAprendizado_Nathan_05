Branches e Hospedagem

Branches

São ramificações de um projeto de software, tendo por objetivo o desenvolvimento paralelo de:

  1. Novas funcionalidades
  2. Correções de erros
  3. Alteração de código

Tipos de branches:

  • Branch Master (Main): Braço principal de um projeto em um repositório.
  • Branch Secundária: Braço secundário de um projeto em um repositório.

Passo a passo:

  • No GitHub Desktop crie um novo repositório.
  • Publique o seu repositório (Ctrl+P).
  • Abra seu repositório no Visual Studio e crie um arquivo index.html.
  • Edite o arquivo index.html como desejar.
  • Realize o commit to main.
  • Até aqui você editou a branch master.
  • Acesse Current Branch e crie um nova branch.
  • New Branch --> Edite o nome como secundaria-01 --> Create branch.
  • Publique sua branch Secundária
  • Se desejar você pode editar a branch secundária.
  • Crie uma segunda branch secundária.
  • Faça as alterações que desejar em sua segunda branch secundária.

Realizando o Merge:

Realizar um Merge é fazer a junção de um Branch secundário em um Branch Master.

  • Selecione a Branch que deseja realizar o Merge --> Menu --> Branch --> Merge into current branch (Ctrl+Shift+M).
  • Selecione quem você deseja realizar o Merge na Branch selecionada.
  • OBS: Se houver algum conflito, clique em Open on Visual Studio e selecione uma opção (Manter as duas alterações, manter a primeira alteração, manter a segunda alteração).
  • Para continuar usando a Branch secundária após o Merge: Menu --> Branch --> Update to main (Ctrl+Shift+U)

Criando uma Branch "Terciaria":

Para isso, basta criar uma Branch apartir de sua Branch Secundária.

Hospedagem

Serve para consolidar o trabalho realizado.

O GitHub possui a funcionalidade do GitHub Pages, que realiza a hospedagem gratuita de repositórios públicos.

Sua URL será: https:/nomeusuario.github.io/repositório

Passo a Passo:

  • Abra seu repositório no GitHub.
  • Settings --> Pages.
  • Source --> None --> Selecione o main --> Save.
  • PRONTO! Basta atualizar a página (F5) e acessar seu site.

Agora que você publicou o seu site, deve tomar alguns cuidados como não editar o Branch Main diretamente, sempre editar um Branch Secundario, testar e depois realizar o Merge.
